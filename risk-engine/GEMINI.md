# Risk Engine: Microstructure Module

Этот модуль отвечает за анализ микроструктуры рынка в реальном времени, используя потоки сделок и обновлений стакана.

## Новые индикаторы

### 1. Order Flow Imbalance (OFI)
Измеряет дисбаланс входящих заявок на покупку и продажу на уровне Best Bid/Best Ask.
- **Расчет:** `Change(Bid Vol) - Change(Ask Vol)` при изменении цены или объема.
- **Интерпретация:** Положительный OFI указывает на агрессивное давление покупателей, отрицательный — продавцов.

### 2. Book Imbalance (Pressure)
Отношение объема заявок на стороне Bid к общему объему в стакане (на глубине 10 уровней).
- **Диапазон:** `-1.0` (полное давление Ask) до `1.0` (полное давление Bid).
- **Значение:** Помогает понять, какая сторона стакана "тяжелее".

### 3. Cumulative Volume Delta (CVD)
Накопленная разница между объемом рыночных покупок и продаж.
- **Реализация:** Считается в реальном времени по потоку сделок.
- **Важно:** Позволяет видеть агрессию рыночных участников.

### 4. VPIN (Volume-Synchronized Probability of Informed Trading)
Измеряет токсичность потока ордеров.
- **Логика:** Разбивает поток сделок на корзины равного объема (Volume Buckets) и считает дисбаланс внутри них.
- **Интерпретация:** Высокий VPIN (> 0.7) сигнализирует о высокой вероятности информированной торговли, что часто предшествует всплескам волатильности или разворотам.

### 5. Liquidity Voids (Пустоты ликвидности)
Идентифицирует ценовые зоны, где в стакане практически нет заявок (ценовые разрывы).
- **Типы:** Gaps в Bid/Ask или "пустой" спред.
- **Значение:** Прохождение цены через такие зоны происходит мгновенно (высокий риск проскальзывания).

## Конфигурация

```typescript
microstructure: {
  tradeWindowMs: 60_000,      // Окно для расчета объемов и дельты (1 минута)
  imbalanceDepth: 10,        // Глубина стакана для анализа давления
  vpinBucketVolume: 100_000, // Объем одной корзины для VPIN ($100k)
  vpinBucketCount: 50,       // Количество корзин в окне расчета
}
```

## Публикация данных

Данные публикуются в Redis Snapshot: `risk:snapshot:microstructure:{exchange}:{symbol}`.
Доступны через API Gateway по каналу `microstructure:{exchange}:{symbol}`.
